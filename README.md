# Excel Smart Parser v20

> Универсальный парсер Excel-файлов с веб-просмотрщиком результатов. Никаких настроек — просто отдаёте файл, получаете данные.

---

## 📋 Возможности

| Функция | Описание |
|---------|----------|
| **Форматы** | `.xlsx`, `.xlsm`, `.xls`, `.xlsb`, `.csv` |
| **Авто-обнаружение** | Находит все таблицы на листе без указания диапазонов |
| **5 источников** | Встроенные таблицы → именованные диапазоны → эвристика → вертикальные → беззаголовочные |
| **Скрытые строки** | Включаются по умолчанию — данные не теряются |
| **Формулы** | Автоматическое предупреждение если ячейки содержат формулы |
| **Стриминг** | Режим `--stream` для огромных файлов без загрузки в RAM |
| **3 формата вывода** | JSON, JSONL, CSV |
| **Кодировки** | Авто-определение для CSV (UTF-8, CP1251, и др.) |
| **Разделители** | Авто-определение для CSV (`,`, `;`, `\t`, `|`, `^`, `~`) |
| **Веб-просмотрщик** | Встроенный HTML-viewer для просмотра результатов в браузере |

## 🚀 Быстрый старт

### 1. Установка зависимостей

```cmd
pip install openpyxl
```

Для полной поддержки всех форматов:

```cmd
pip install openpyxl xlrd pyxlsb chardet tqdm xlwt
```

> **xlwt** нужен только для тестов. **xlrd** — для `.xls`. **pyxlsb** — для `.xlsb`. **chardet** — авто-кодировка CSV. **tqdm** — прогресс-бар.

### 2. Запуск

```cmd
python excel_smart_parser.py файл.xlsx
```

Результат: `файл_parsed.json` в той же папке.

### 3. Просмотр результатов

После парсинга откройте файл через встроенный веб-просмотрщик:

```
excel_viewer/index.html
```

Перетащите JSON-файл в браузер — таблицы, поиск, сортировка, фильтрация.

### 4. Несколько файлов

```cmd
python excel_smart_parser.py отчет1.xlsx отчет2.xlsx отчет3.xlsx
```

## 📖 Все команды

### Сводная таблица CLI-аргументов

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `files` | *(обязательно)* | Один или несколько файлов для парсинга |
| `--out-dir DIR` | Директория файла | Куда сохранить результат |
| `--format FMT` | `json` | Формат вывода: `json`, `jsonl`, `csv` |
| `--sheet NAME` | Все листы | Обрабатывать только указанный лист |
| `--header-threshold N` | `0.4` | Порог score для заголовка (0.0–1.0) |
| `--min-data-cells N` | `2` | Мин. непустых ячеек для строки данных |
| `--skip-hidden` | Выключено | Пропускать скрытые строки и колонки |
| `--stream` | Выключено | Потоковая запись (для больших файлов) |

### Примеры

```cmd
# Только конкретный лист
python excel_smart_parser.py отчет.xlsx --sheet "Расчет"

# Результат в конкретную папку
python excel_smart_parser.py отчет.xlsx --out-dir C:\Результаты

# CSV — каждая таблица в отдельном файле
python excel_smart_parser.py отчет.xlsx --format csv --out-dir C:\Результаты

# JSONL — по одной JSON-строке на запись
python excel_smart_parser.py отчет.xlsx --format jsonl

# Потоковая запись для огромных файлов
python excel_smart_parser.py огромный.xlsx --stream --out-dir C:\Результаты

# Настроить чувствительность
python excel_smart_parser.py файл.xlsx --header-threshold 0.3 --min-data-cells 3
```

## 🌐 Веб-просмотрщик (Excel Viewer)

Встроенный HTML-просмотрщик для удобного чтения распарсенных данных. Не требует сервера — работает полностью в браузере.

### Расположение

```
excel_viewer/index.html
```

### Как использовать

1. Распарсите Excel-файл парсером → получите `.json`
2. Откройте `excel_viewer/index.html` в браузере
3. Перетащите JSON-файл на страницу или нажмите «Загрузить»

### Возможности просмотрщика

| Функция | Описание |
|---------|----------|
| **Drag & Drop** | Перетаскивание файла на страницу или клик для выбора |
| **Вкладки** | Переключение между таблицами из разных листов Excel |
| **Поиск** | Мгновенный поиск по всем столбцам с подсветкой совпадений |
| **Сортировка** | Клик по заголовку столбца (▲ по возрастанию, ▼ по убыванию) |
| **Тёмная тема** | Интерфейс в стиле GitHub Dark |
| **Офлайн** | Не требует сервера, все данные обрабатываются локально |
| **Производительность** | Рендеринг до 2000 строк одновременно, остальные — через поиск |
| **Без зависимостей** | Один HTML-файл, никаких npm, CDN или библиотек |

### Скриншот

При открытии отображается экран загрузки с drag-and-drop зоной. После загрузки — таблицы с вкладками, строкой поиска и сортируемыми колонками.

## 🏗 Архитектура

### 5 источников таблиц (от точных к эвристическим)

```
1. native_table  — встроенные Excel-таблицы (Ctrl+T)
2. named_range   — именованные диапазоны
3. heuristic     — эвристический поиск по заголовкам
4. vertical      — вертикальные таблицы (заголовки в колонке A)
5. headerless    — матрицы без заголовка (колонки = A, B, C...)
```

Каждый источник дополняет предыдущие. Перекрытия предотвращаются через `used_rows`.

### Адаптеры форматов

```
SheetAdapter (abc.ABC)
├── OpenpyxlAdapter  → .xlsx, .xlsm, .xltx, .xltm
├── XlrdAdapter      → .xls
├── PyxlsbAdapter    → .xlsb
└── CsvAdapter       → .csv
```

### Определение заголовков

Score-based система (0.0–1.0):

| Тип ячейки | Вклад |
|------------|-------|
| Текст 2–60 символов | +1.0 |
| Однобуквенный текст ("A", "№") | +0.7 |
| Дата | +0.5 |
| Строковый год ("2024") | +0.3 |
| Число-день (1–31 подряд) | +0.5 |
| Число (обычное) | −0.2 |
| Длинный текст (>60 символов) | −0.5 |

Порог по умолчанию: **0.4**

## ❓ Troubleshooting

### `❌ pip install openpyxl`
Не установлена обязательная зависимость. Запустите:
```cmd
pip install openpyxl
```

### `❌ pip install xlrd  # для .xls файлов`
Пытаетесь открыть `.xls` без xlrd. Запустите:
```cmd
pip install xlrd
```

### `UserWarning: обнаружены формулы. data_only=True вернёт None`
Файл содержит формулы, которые не были пересчитаны при последнем сохранении. Откройте файл в Excel, сохраните, и запустите парсер снова.

### Таблицы не находятся
Попробуйте понизить порог заголовков:
```cmd
python excel_smart_parser.py файл.xlsx --header-threshold 0.3
```

### Слишком много мусорных таблиц
Повысьте порог или увеличьте минимум ячеек:
```cmd
python excel_smart_parser.py файл.xlsx --header-threshold 0.6 --min-data-cells 3
```

### `MemoryError` на больших файлах
Используйте потоковый режим:
```cmd
python excel_smart_parser.py большой.xlsx --stream --out-dir C:\Результаты
```

## ⚠️ Ограничения

- **VBA-макросы** (`.xlsm`) — игнорируются, парсится только содержимое ячеек
- **Диаграммы и графики** — не извлекаются
- **Условное форматирование** — не учитывается
- **Защищённые листы** (с паролем) — не открываются
- **Очень большие формулы** в `.xlsb` — могут не распарситься корректно через pyxlsb

## 🐛 История версий

### v20 (текущая)

- Эвристика корректно разделяет таблицы по заголовкам после пустых строк
- Скрытые колонки фильтруются в native tables и named ranges (`--skip-hidden`)
- NaN/Inf не создают призрачные строки в выходных данных
- BOM (UTF-8) не утекает в первую ячейку CSV
- Научная нотация парсится в CSV (`1e5`, `-1E10`, `2e-3`)
- CSV streaming cleanup при ошибке парсинга
- Вертикальные таблицы корректно блокируют used_rows
- Bounds checking во всех адаптерах (Openpyxl, Csv)
- `_detect_dtype`: boolean порог `>= 0.5` (было `> 0.5`)
- 257 unit-тестов (+107 новых: большие данные, edge cases, streaming)

### v19

- Эвристика запускается ВСЕГДА (не только при пустом all_tables)
- `_build_rows()` — единый метод чтения ячеек без дублирования
- `--include-hidden` заменён на `--skip-hidden` (старый флаг не работал)
- XlrdAdapter: убран повторный import внутри cell()
- PyxlsbAdapter: named_ranges за один проход без повторного открытия файла
- `_serialize`: обработка datetime.time и timedelta
- `_detect_dtype`: распознавание boolean значений
- CSV: пробел убран из разделителей
- `ExcelParser.__init__`: валидация параметров
- 150 unit-тестов

### v18.1

- `SheetAdapter` → `abc.ABC` с `@abstractmethod`
- `except: pass` → `warnings.warn` (~6 мест)
- Двойное открытие файла устранено (один scan + один full open)
- `_detect_dtype` из кэша, без повторных `adapter.cell()`
- `is_day_header` с защитой от `nan`/`inf`
- `max_empty_streak` увеличен с 3 до 50
- `skip_hidden` по умолчанию `False`
- Headerless источник запускается всегда
- `native_tables` — совместимость с разными версиями openpyxl

### v17

- Распознавание заголовков с числами-днями (1–31)
- Убран разрыв таблицы по маркер-строкам
- Скрытые строки включены по умолчанию

### v11–v16

- Дедупликация заголовков
- Кэш видимых строк/колонок
- Потоковый writer
- Percent-детекция по >30% значений
- Именованные диапазоны для .xls
